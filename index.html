构造函数、实例、prototype

在介绍封装和集成之前，需要了解 JS 对象的三个概念： 构造函数、实例 和 prototype。

以生产某个型号A的机器人为例。生产机器人我们首先需要一个设计好的模具(构造函数，java 里叫做类)，才能够按照这个模具成产出一个一个机器人(实例)，你可以给其取名 a、b、c、d(对象变量名，实际是指针)。另外，你可以根据匹配磨具生产出唯一的一套配件外设(prototype,即原型)。当你买了一个机器人 a 的时候(创造了一个对象 a)，你可以说对象有模具中的功能，也有提配件供外设提供的功能。

在 JS 中，要实例化某个对象，先要声明一个构造函数(模版，类),实例化一个实例(对象)就是按照模版中的代码在新开辟一块内存，然后让你的变量名指向这块内存。prototype(原型)是所有实例和构造函数共享的一套方法和属性，有一块自己独立内存，构造函数的内存和实例的内存中都有一个指针指向 prototype的内存。

总结起来：

  1. 构造函数有一块内存，存储模版代码；
  2. 每个实例是按照构造函数代码生成的一块自己的内存，即已复制；
  3. prototype有自己的一块内存，内存中的实例和方法都是可以直接引用的。构造函数和实例中有指针同时指向这一块内存，感觉像每个实例中都有prototype中的属性和方法，但是大家其实用的都是同一个东西；
  4. 构造函数和实例中指向 prototype（原型）的指针就叫 prototype;另外，prototype 中有一个指向构造函数的指针 constructor,把构造函数从这么多指向他的内存块中区分出来。

JS的继承

继承有两种：接口继承和实现继承。因为 JS 不支持接口，所以 JS 的继承都是实现继承。

JS 的继承主要有三种方法：原型链继承、构造函数继承、原型链和构造函数的组合继承。

但是首先需要了解几个概念。

原型链

在之前的介绍中，一个对象实例的 prototype 属性指针是指向这个实例的原型的。在使用的时候，实例先在自身查找属性和变量，不存在的话就通过 prototype 指针到原型上查找。

但是，当我们把一个实例 A 的 prototype 指针修改指向到另一个类型的对象实例 B，而 B 又指向他的原型。这样，我们的查找从原先的 实例 -> 原型 变成了 实例A -> 实例B -> 实例B的原型,这条链就2节变成了3节。当然，我们也可以根据实际需求，让 B 的 prototype 属性指针指向实例 C,让 C 指向实例 D … 让这条链根据需求可长可短。这条链就叫做 原型链。

使用原型链继承

使用原型链继承正如上述的原型链的概念，在链的头部的实例 A 继承了链后的所有属性和方法。

原型继承存在一个问题：父类构造函数中私有变量会变成子类中所有实例的共有变量。

使用构造函数继承

使用构造函数继承的策略很简单，就是在子类的构造函数中调用需要继承的父类的构造函数。由于构造函数需要在特定环境中执行，所以需要使用到 call()、apply()这两个函数来改变执行环境 this。

构造函数继承存在的问题是：只能继承父类构造函数中的方法和属性，无法继承父类中的原型。

原型链和构造函数组合继承

组合继承是结合原型链和构造函数继承的有点，使用原型链继承父类的原型，使用构造函数继承构造函数内的变量和属性。

使用原型链继承，会继承父类原型和构造函数中的所有属性和方法；而使用构造函数，会将父类构造函数的方法调用到子类的构造函数中，由于实例是先查找构造函数中，在查找原型链中的属性和方法，所以使用原型链继承的那一部分父类构造函数的属性方法会被使用构造函数继承的覆盖掉，从而避免了单纯使用原型链继承而导致父类构造函数的属性方法被共享的问题，但同时又保障了父类构造函数属性方法的继承。

基本包装类型概述

ECMAScript 5标准中，JS 有6种数据类型，Number、Boolean、String、undefined、null和 Object。Object是引用类型，其余的是基本类型，不具备对象特征(null 是空指针，可以看做 Object)。

但是在实际在读写 Number、String、Boolean 类型时，系统会动态为这三种类型包装上一层对象类型的壳，可以让我们像对象一样操作它们，在读写完后又会卸下这层壳（PS：JS中所有变量都是对象这种说法是不正确的）。这个加壳的过程如下：

  1. 读写到 Number、String、Boolean这三个基本类型变量值时，系统自动为其创建一个相应的包装对象(就是一个包含特定方法属性的对象)，包装上去；
  2. 使用包装对象提供的方法和属性进行操作，让人感觉是在使用对象一样；
  3. 当读写语句执行完，系统自动卸载掉、销毁包装对象，恢复成基本类型。

包装对象(即”壳”)首先是一个对象，包含所有对象都包含的 construcotr、hasOwnProperty()、isPrototypeOf()、prototypeIsEnumerable()、toLocalString()、toString()、valueOf() 七个方法基本方法。
