JS匿名函数和闭包-读书笔记

Posted on 2016-06-08 • In JavaScript Note

匿名函数

匿名函数介绍

如上所示，JS函数的声明有两种形式，直接定义和将函数赋值给一个变量。在第二种中，function(){}这种函数称之为匿名函数。之前我们提到过函数名就是一个指向函数内存区域的指针，所以单独声明 function(){}这种形式是没有指向这个函数的指正的，程序运行过这段代码即回收掉这段内存。但是可以用把它复制给一个变量，让这个变量充当这个指针。

单纯 function(){}形式匿名函数的执行可以使用上述第三个例子的方式，(function(arg){})(argEg)直接给匿名函数传参让其执行。

块级作用域

在 ES 5标准中，是没有块级作用域的。但是我们可以使用匿名函数的特征，构造出跨级作用域。

闭包

作用域链

在 JS 中,一个函数可以访问这个函数内部定义的变量,也可以访问这个函数外部环境中定义的变量. 当这个函数A的直接外部环境是函数 B, B 的直接外部环境是函数 C…,则在 A 中使用一个变量的时候,其查找顺序是 A->B->…全局环境.这个查找顺序就叫做作用域链。

作用域链的数据结构其实就是表和指针。A 把其作用链域放在一个表里,表的每一项都是指针,分别指向 A 中定义的变量、B 中但不是 A 里定义的变量…全局里定义的变量。

闭包概念

向上述，一个函数中包含另一个函数，函数 two中使用函数 one 中定义的变量的情况就是闭包。闭包的定义为：有权访问另一个函数作用域中变量的函数。

闭包的原因就是作用域链。

闭包的有一个特性，即当调用了函数 two()，则函数 two()中引用的变量 a 会一直存在于内存中，和 two()的生命周期一样，即使之前在正常情况下 one()的内存要被消除，a 也不会被消除。这是因为在作用域链中，one()要被消除，则 one 指向变量 a 的指针会被消除，但是之前有调用 two(),two()的生命周期没有结束，而 two()的作用域链中有指针指向 a，所以 a 不会被垃圾回收机制回收掉。

闭包的这个特性有很多应用场景，例如定义私有变量等；但是也存在缺点：占用大量内存。
