这是一篇前端资源收集博文

发表于 2016-06-03   |   分类于 滕王阁   |  

写在前边

这是一篇关于前端资源和学习建议的博文。主要包含两个部分，学习建议部分，主要讲的是笔者学习前端到现在的学习感悟(踩坑无数），可以作为前端入门参考，仅作参考;资源收集部分集中了一些优秀的前端资源，这些资源多数来自于我的游览器书签，多数是读过或了解过的，而不是 “先 mark 一下”的内容。

学习建议

前端入门

建议新手入门先花一周时间把 W3cschool 上的 HTML、CSS、JavaScript三部分的内容全面看一遍，相关代码都敲一敲，尽量把主要 API 都背下来~

然后找几个简单的网页，模仿着写一遍~在写的过程中熟悉HTML、CSS、JS 是如何协调在一起工作的和其相关语法、API，并熟悉游览器中开发者工具的使用(火狐的 firebug、Chrome的开发者工具)。

然后就是边阅读学习边动手写代码提高的过程了。进阶学习部分推荐 MDN 开发者社区上的 JS 教程和 JS高级程序设计，一定要系统的学习一遍 JS。

习惯

建议养成以下良好的习惯：

  1. 盲打，掌握正确的指法；
  2. 掌握尽量多的快捷键，减少使用鼠标的时间，这会提高你的打字速度；
  3. 学习自动化，使用脚本、自动化工具等构建你的自动化工程；
  4. 找一好的 JS 教程，例如 JS高级程序设计或者犀牛书，尽量把内容在理解后都背下来，这样就不用编写便查 API了；
  5. 学英文，这样就可以直接看外文文献而不是二手资料了；
  6. 坚持写博客；
  7. 在 GitHub 上开启自己的业余项目；
  8. 有自己的知识的输入-管理-输出系统，输入可以是看书、看博客、游览网页内容等形式，管理是一个知识的吸收整理过程，可以是思维导图+印象笔记这样一个系统，输出是最重要的，写博客和写程序都是一种输出。只有三者均衡都做的不错时，学到的东西才会越多。
