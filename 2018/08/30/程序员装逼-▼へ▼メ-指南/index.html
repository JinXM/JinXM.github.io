程序员装逼(▼へ▼メ)指南

2018-08-30 Alucard

其实程序员，是实际应用中和科研结合特别紧密的一个行业。

所以，程序员的外(zhuang)化(bi)工程就和知识管理(PKM)模型特别像。

PKM = Personal Knowledge Management

PKM 有三个步骤：

  1. 知识的获取；
  2. 知识的内建；
  3. 知识的输出；

专业知识和软实力知识的学习，是后期内建和输出的基础。学习的过程包括知识、感受的获取，然后理解，初步建立记忆体系。

内建，在初期的理解消化之后，多数是在实践中的进行应用、提炼总结获取的高价值知识、观念。

知识的输出形式则是多样的，日常工作技能的输出、写书写博当网红、国内新技术的布道…只要你前两步做的足够强大，输出的形式只有你想不到的。

这里以一个毕业工作2-3年，成长速度普通的小码农作为标本，探讨一些有效可行的外(zhuang)化(bi)手法。

因为想法是作为可操作的指南 List，这里只列出小数量的 Item，也作为自己的实践地图来执行。

lv0~9：

  • 搭建自己的博客。以内容为王，持续记录一些有质量东西，随着时间的积累，将会有客观的收获。
  • 适当参与一些论坛、大会，了解行业内一些新动态和高level 们是如何装X的

lv10~19:

  • 搭建自己的明星 Github 主页
    • 有一定质量的代码，star 数和 fork 数当然是比较直观的反映
    • 参与一些组织，并有一定量的代码贡献
    • commit 小表格虽然不能绝对代表什么，但是满满的绿色拿来吓吓人还是不错的（hahahahha~
  • 日常注重自己的表达，尽量尝试以 说人话 + 严谨简洁 描述
    • 说人话是把自己当做产品经理和销售，即使对方是非自己领域或者小白，也能比较容易的听懂(一些)
    • 严谨是专业性的体现，简洁则是能让自己在别人的耐心之内把话说完
    • 小技巧：刚开始可以把话说慢
  • 开始在一些公共网络写(分享)有价值博客和观点，让圈内的人认识和认同你
  • 有一些不错的公司经历，比如一家技术氛围很棒的初创公司、比如国内BAT、国外一些不错公司的任职经历…

lv20~30：

  • 靠产品说话。如果能有自己主导(之一)的优秀产品真是太棒了！可以作为参与者也是很不错的
  • 作为嘉宾，参与一些论坛、大会、沙龙
  • 去一些更佳不错的公司，实现自己的愿景
