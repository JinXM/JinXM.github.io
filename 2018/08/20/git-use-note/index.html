命令功能索引

创建 git 项目

git init

作用: 将一个目录初始化为 git 项目. create an empty Git repository or reinitialize an existing one

eg: 本地建了一个项目 my-project.

这样, my-project 就变成了可以被 git 管理的项目了.my-project 目录下会生成一个.git 的文件, 里边存放 git 版本的快照.

git clone

作用: clone 远程项目到本地. clone a repository into a new directory

eg: clone 远程项目的某个分支

基本的快照

git 本地仓库的三种状态

  • 工作目录

实际文件的持有者,也就是系统中你的项目文件目录.

  • 暂存区(Index)

一个缓存区, 里边缓存项目代码相对于上次提交版本的变动.添加到 Index的文件会被追踪.

  • HEAD

指向最后一次提交的结果.也就是 Index 里的”上次提交的版本”.

基本套路

使用 git add 添加需要追踪的新文件和待提交的更改 -> 然后使用 git status 和 git diff 查看有何改动 -> 最后用 git commit 将你的快照记录.

git status

作用: 查看项目文件所对应的状态. show the working tree status

加参数-s是以精简的形式列出文件状态.

git add

作用: 将变动文件从 工作目录 -> Index. 将变更保存到缓存目录. add file contents to the index

git commit

作用: 将代码从Index -> HEAD, 从缓存生成实际快照. 但是还没提交到远程. record changes to the repository

可以通过添加 -a 来跳过 git add. -a 表示默认 git add 所有已经被跟踪的文件.

查看某个commit 的内容

分支和合并

基本套路

git branch (branchname) 或 git checkout -b (branchname)创建新分支 -> git checkout (branchname)切换分支.当你切换分支的时候，Git 会用该分支的最后提交的快照替换你的工作目录的内容 -> 使用 git merge 来合并分支。你可以多次合并到统一分支， 也可以选择在合并之后直接删除被并入的分支.

git branch

作用: 新建, 删除, 列出分支. list, create, or delete branches

git checkout

作用: 切换分支. checkout a branch or paths to the working tree

加上参数 -b 可以直接新建一个分支,并切换到这个分支.

git merge

作用: 将分支合并到你当前的分支. join two or more development histories together

eg: 将 hotfix 分支上的更改内容合并到 master 分支上.

git diff

作用: 对比版本, 文件之间的代码变更. how changes between commits, commit and working tree, etc

git log

作用: 查看快照操作日志, 即所有 git commit 操作的日志. show commit logs

git tag

作用: 给历史记录中的某个重要的一个快照打上标签. create, list, delete or verify a tag object signed with GPG

如果你达到一个重要的阶段，并希望永远记住那个特别的提交快照，你可以使用 git tag 给它打上标签。 该 tag 命令基本上会给该特殊提交打上永久的书签，从而使你在将来能够用它与其他提交比较。 通常，你会在切取一个发布版本或者交付一些东西的时候打个标签。

eg: -a参数表示选项意为“创建一个带注解的标签. 这里创建一个 ‘publish’ 的 tag, 表示这个快照是用来发布使用的.

更新和提交

基本套路

使用 git fetch 更新你的项目 -> 使用 git push 分享你的改动 -> 你可以用 git remote 管理你的远程仓库.

git remote

作用: 罗列、添加和删除远端仓库别名. Git可以拥有多个远端仓库变 —— 你可以拥有一些只读的仓库，另外的一些也可写的仓库。

git pull & git fetch

git fetch作用: 将你的仓库与远端仓库同步，提取所有它独有的数据到本地分支以合并或者怎样. download objects and refs from another repository

git pull作用: = git fetch + git merge

git push

作用: 提交本地快照到远程仓库. update remote refs along with associated objects

gitignore

作用: .gitignore 配置文件用于配置不需要加入版本管理的文件(即需要被忽略的文件).

用法: 在 git 项目根目录下创建.gitignore 文件.通常使用 git init 的时候,可以选择自动生成这个文件.

语法:

  • 以斜杠“/”开头表示目录; eg: /node_modules
  • 以星号“”通配多个字符; `eg: .exe`
  • 以问号“?”通配单个字符;
  • 以方括号“[]”包含单个字符的匹配列表;
  • 以叹号“!”表示不忽略(跟踪)匹配到的文件或目录;

git 管理策略

例: 比如你要在一个组织org1的项目fly中开发.

方案一

  1. fork fly 到自己的 github, my/fly.
  2. git clone my/fly 到本地. 为 master.
  3. 建立分支 git checkout -b develop , 作为日常开发分支.
  4. 建立远程组织仓库别名: git remote org orgUrl

下拉更新:

  1. 原则: 让自己本地& my 下的 master和 org 下的保持一致; 自己的 develop 和 my/develop 和开发进度保持一致;
  2. 下拉 org 稳定发布的更新代码: git fetch org master;
  3. 同步到本地 master ,my/master, 本地 develop: git checkout master, git merge org/master, git push orign master, git checkout develop, git merge org/master;

提交代码:

方案二

  1. fork 到自己的 github.
  2. git clone 到本地.
  3. 本地建立分支 develop, 和 org 上稳定发布的代码保持一致;
  4. 基于本地 develop, 本地新建立分支 feature_x, feature_y…, 每个分支用于一个功能或 issue 的开发;
  5. feature 分支更新代码: 直接merge org 上稳定的代码;
  6. feature 提交:
    1. 测试完后 merge 到本地develop, 再有 develop push 上去;这样 my 下只有 master 和 develop 分支.
    2. 测试完稳定的 feature 直接 push 到 my. 这样 my 下也可见很多 feature 分支.
