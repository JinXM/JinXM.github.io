hexo搭建博客填坑半记录

2018-09-01

貌似自己每年都要对博客这个坑踩一回，其实也就 UI 视觉什么的缝缝补补，作为每年练手 CSS 的样本（虽然总是越改越丑…

工作久了，越来越觉得不应该是“写博客”而应该是“写博客”，毕竟是内容为王的东西。

所以打算填完这个版本的坑后，以后就不打算改什么样式了（老了，改不动了…

这个版本，主要是做了一下优化或者说设计：

  1. PC 端屏幕 Width 大于 Height，所以采用 左侧 sidebar + 右侧main的布局，尽量是阅读主题的main 比例接近于正方形甚至 Height > Width。这么做的原因是，让阅读部分的 main 更接近于传统纸质阅读书籍的比例。
  2. 移动端，主要考虑到微信圈的分享。所以样式会尽量接近于公众号的样式。然后文章内阅读，只在文章底部放一个站点链接，不再顶部展示任何导航信息，让文章的阅读体验更纯粹。
  3. 在字体样式上做了一些琢磨，不再是以字体好看为目的，而是以适合轻负担阅读的字体作为目的。文章的类型分为2类，生活博和技术博。生活博基本全是表述性文字，更注重于文字本身含义的美感和表达出来的情感；技术博会有大量代码、专业词汇、表格列表之类的，需要更丰富和密集的表达。所以对于两种类型的博客，文字样式上做了一些细微的区别，以更好的展示两者的特点。
  4. 会有一些单独的页面，比如图片展示、Plan 打卡之类的，用于特殊需求的展示。
