二分查找T(logn)

二分查找可以这么理解。

A问你，他写了一个在0-100中间的数字x(=32)，让你猜。
你可以问比某个数字大还是小，如何尽快猜对。

你：比50大还是小？
A：小。
你：比25大还是小？
A：大。
你：比37大还是小？
A：小。
你：比19大还是小？
A：大。
你：比28大还是小？
A：大。
你：比33大还是小？
A：小。
你：比31大还是小？
A：大。
你：是32吗？
A：是。

即：每次询问，查询范围缩小为原来的一半length/2，当n 次询问后，查询范围变成1，即可得到结果。也即length 除以n次2，也即2^n=legnth，所以n=log(length)。

低级排序

主要是选择排序和插入排序，时间复杂度都为n^2。

选择排序是将数组分为左右两部分，左侧是已经排序好的，右侧是乱序。从右侧选出最小的，放到左侧，然后左侧慢慢增加，右侧慢慢减少，直到全部变成左侧。是左不动右动。

插入排序，则相反是左动右不动。右边和左边相邻的数字扔进左边去排序，等左边排好再把下一个数字扔进去，直到全部变成左边。

还有插入排序的优化版本，希尔排序。时间复杂度突破了n^2，具体级别和排序使用的参数有关。

希尔排序

希尔排序是插入排序的改进。

插入排序的时间复杂度是T(n^2)，希尔排序突破了n^2的数级。

插入排序的时间复杂度和原始数组结构关系很大，比如最小值在最右边，就需要挪动len-1的长度到最左边。而且数组长度越大，挪动越多。

所以如果可以做到：1）一次挪动很长的步幅，比如直接从最右边一步挪到最左边，能节省很多；2）将数组切割成x部分，每个小部分内插入排序，单次则会变成(n/x)^2 * x，也是比较节省的。

综上，希尔排序设定一个大步幅step，比如step=len/2，那index 的分组为[0, len/2], [1, len/2 + 1]…[len/2 - 1, len -1],每组作为一个子插入排序数组进行排序。
然后再将step = step/2=len/4，获得新的子插入排序数组进行排序。
依次类推，当step 为1时，即完成数组的排序。
